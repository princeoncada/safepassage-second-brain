### Patch 4.18.1 — GLEN QA Tip Wording Fix
Corrected the GLEN registered-tag QA tip from "license plate or RFID
tag registered in system" to "visitor already confirmed in SP Guard
with active access." Clarified during session: RFID is physical
hardware that auto-opens gates; "registered" in SP Guard context means
the visitor's identity was already verified and access was granted in
a prior interaction.

### Patch 4.18.2 — Prevent DeepSeek Inline Sources Block
Added explicit rule to answer_from_context.md instructing DeepSeek
not to generate any Sources section in its answer body. Fixes
duplicate sources output where DeepSeek was embedding a **Sources:**
block mid-answer AND the CLI was printing its own Sources block.

### Patch 4.18.3 — Restore Clean CLI Citation Display
Updated `cited_source_ids()` regex in answer_vault.py from
`r"\[(\d+)\]"` to `r"\[(?:Source )?(\d+)\]"` to match both [N] and
[Source N] inline citation formats. The 4.18.2 change caused DeepSeek
to switch to [Source N] format, breaking the CLI citation parser and
triggering verbose debug fallback output.

## Key Decisions Made

- GLEN QA tip classified as type: qa_rule (advisory), not post_order.
  Post orders are higher authority and override QA tips if they conflict.

- "Registered" in the GLEN tip context means visitor confirmed in
  SP Guard with active access — not RFID hardware. RFID auto-opens
  gates and bypasses the kiosk entirely.

- Call flow output format: Option B (merged rewrite). Community-
  specific modifications are integrated naturally at each step, not
  shown as a separate "modifications" section.

- External senior engineering review incorporated. Confirmed the repo
  is at Stage 4 (Trust hardening) entering Stage 5 (Architecture
  stabilization). Main smell: separation of concerns not yet enforced.

- Development priority order confirmed:
  1. Operational trust (audit log — done in 4.19.0)
  2. Speed (model preloading — done in 4.20.0)
  3. Handoff readiness (Phase 4.21.0)
  4. Architecture safety (Phase 4.22.0)
  5. Developer scalability (Phase 4.23.0)

- Audit log stores full answer text (not preview). Company is
  compliance-heavy; full text is required for dispute resolution.
  Log file is append-only, gitignored, never read by AI pipeline.

## Key Decisions Made This Session

- Incremental indexing uses mtime scan (60s window) to detect newly written
  vault files after ingestion rather than passing file paths from the
  ingestion script directly.
- Streaming uses requests (already a transitive dep) not httpx; urllib
  call_deepseek() kept intact for CLI path.
- Conflict detection uses anchor token approach not topic key similarity;
  similarity threshold approach was too sensitive to sentence structure.
- CANCEL preferred over NO for wizard cancellation; backend accepts both
  NO and CANCEL for backwards compatibility.
- Quick reply hints are pipe-only (no backend changes); genuine Action
  Function button UX deferred to future phase.
- Confidence/reason/warnings footer kept in pipe output for now (personal
  use case); will be cleaned up for production.
- Phase ordering confirmed: 4.14 incremental → 4.15 streaming → 4.16
  conflict+wizard → 4.17 pipe UX.
- Future phases tracked in docs/FUTURE_PLANS.md going forward.
